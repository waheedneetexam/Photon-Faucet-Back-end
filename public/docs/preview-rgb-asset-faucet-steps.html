RGB Asset Faucet Steps

These steps explain how the public RGB faucet on faucet.photonbolt.xyz/rgb-faucet.html fits into the PhotonBolt preview and wallet workflow.

1. Fund the wallet with RegTest BTC first

Use the Bitcoin RegTest faucet before using the RGB faucet. The wallet needs BTC and RGB-ready UTXO state before more advanced RGB flows work reliably.

2. Import the asset from the live registry

Open the live asset registry and import the contract into Photon Wallet. The preview site now supports direct wallet import from the registry when the extension is connected.

3. Decode or paste the RGB invoice

The public RGB faucet can decode invoices and show the asset name from the registry even when the invoice itself only contains the contract identifier.

4. Claim a supported faucet asset

Use the faucet claim flow only for assets that are actually funded in the faucet node inventory. Registry visibility alone does not mean the faucet can distribute the asset.

5. Refresh Photon Wallet after settlement

After the faucet completes the claim, return to Photon Wallet and refresh the asset list so the live RGB node balance updates the spendable and pending amounts.

  • If the asset is missing, import it from the registry first.
  • If pending values remain stale, use the RGB node refresh tools or wallet reload.
